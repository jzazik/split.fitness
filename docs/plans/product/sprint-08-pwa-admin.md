# Sprint 8: PWA и административная панель

## Проблема / Краткое описание

Приложение должно работать как нативное на мобильных устройствах через PWA технологию. Администраторам нужны инструменты для модерации тренеров, мониторинга платформы и поддержки пользователей.

## Ценность для пользователя

**Для атлетов и тренеров:** Установка приложения на телефон, быстрый запуск с домашнего экрана, работа в полноэкранном режиме, ощущение нативного приложения.

**Для администраторов:** Полный контроль над платформой, возможность модерации, просмотр метрик, управление проблемными ситуациями.

**Для бизнеса:** Увеличение retention через PWA, operational excellence через админку, качество контента через модерацию.

## Скоуп спринта

### Включено:

**PWA функционал:**
- Установка на домашний экран (iOS/Android)
- Полноэкранный режим работы
- Splash screen при запуске
- Иконки приложения
- Offline страница-заглушка
- Кеширование статики

**Админ-панель (Filament):**
- Модерация профилей тренеров
- Просмотр всех пользователей
- Мониторинг тренировок
- Просмотр платежей и возвратов
- Dashboard с метриками
- Управление справочниками (города, спорт)

### Исключено:
- Push-уведомления
- Offline-режим с полным функционалом
- Background sync
- Экспорт данных
- Массовые операции
- API для внешних систем
- Роли администраторов

## Критерии приёмки

### Установка PWA

**Дано:** Пользователь открыл сайт на мобильном устройстве
**Когда:** Использует сайт несколько раз
**Тогда:**
1. **На Android:**
   - Появляется баннер "Добавить на главный экран"
   - После установки иконка появляется на домашнем экране
   - Запуск открывает приложение без браузерных элементов
2. **На iOS:**
   - Показывается подсказка как установить через Share → Add to Home Screen
   - После установки работает в полноэкранном режиме

### Работа установленного PWA

**Дано:** Пользователь установил PWA
**Когда:** Запускает с домашнего экрана
**Тогда:**
- Показывается splash screen с логотипом
- Приложение открывается в полноэкранном режиме
- Нет адресной строки и элементов браузера
- Статус-бар соответствует цветовой схеме
- При отсутствии сети показывается offline-страница

### Модерация тренеров (админ)

**Дано:** Администратор в панели управления
**Когда:** Открывает раздел "Модерация тренеров"
**Тогда видит:**
- Очередь профилей на модерации
- Для каждого: фото, ФИО, био, виды спорта
- Кнопки: "Одобрить" / "Отклонить"
- При отклонении - поле для указания причины
- Фильтры: Ожидают / Одобрены / Отклонены

### Dashboard администратора

**Дано:** Администратор открывает главную админки
**Когда:** Загружается dashboard
**Тогда видит виджеты:**
- Количество пользователей (атлеты/тренеры)
- Активные тренировки на сегодня
- Сумма платежей за период
- Очередь модерации
- График регистраций
- Последние платежи

### Управление тренировками (админ)

**Дано:** Администратор в разделе "Тренировки"
**Когда:** Просматривает список
**Тогда может:**
- Видеть все тренировки с фильтрами
- Отменить тренировку (с уведомлением участникам)
- Посмотреть список участников
- Изменить статус
- Видеть связанные платежи

### Мониторинг платежей (админ)

**Дано:** Администратор в разделе "Платежи"
**Когда:** Открывает список транзакций
**Тогда видит:**
- Все платежи с статусами
- Детали: кто, когда, за что, сколько
- Raw данные от Тинькофф (для дебага)
- Возможность инициировать возврат вручную
- Фильтры по статусам и датам

### Управление пользователями (админ)

**Дано:** Администратор в разделе "Пользователи"
**Когда:** Открывает профиль пользователя
**Тогда может:**
- Видеть всю информацию профиля
- Заблокировать/разблокировать аккаунт
- Сбросить пароль
- Посмотреть историю действий
- Видеть связанные тренировки и платежи

## UX заметки

### PWA характеристики:
- **App-like:** Навигация и взаимодействие как в нативном приложении
- **Splash screen:** Брендированный экран загрузки
- **Иконки:** Адаптивные под разные устройства
- **Theme color:** Соответствие бренду
- **Orientation:** Portrait для мобильных

### Offline страница:
- Иллюстрация "Нет соединения"
- Текст: "Проверьте подключение к интернету"
- Кнопка: "Попробовать снова"
- Кеш последних просмотренных страниц (если возможно)

### Админ-панель (Filament):
- Стандартный UI Filament
- Тёмная тема (опционально)
- Responsive таблицы
- Batch операции где нужно
- Экспорт в Excel (будущее)

### Подсказка установки iOS:
```
Чтобы установить приложение:
1. Нажмите кнопку "Поделиться" ↗
2. Выберите "На экран Домой" ⊕
3. Нажмите "Добавить"
```

## Валидация и права доступа

### PWA требования:
- HTTPS обязателен
- Valid manifest.json
- Service worker зарегистрирован
- Иконки минимум 192x192 и 512x512
- Start URL корректный

### Админ доступ:
- Только role=admin
- Отдельный URL /admin
- Двухфакторная аутентификация (будущее)
- Audit log всех действий
- IP whitelist (опционально)

### Модерация:
- Нельзя одобрить без проверки обязательных полей
- История всех решений сохраняется
- Email уведомления автоматические
- Возможность повторной модерации

```
Причина: [указанная админом причина]

Что делать:
1. Внесите необходимые изменения
2. Отправьте профиль на повторную проверку

Редактировать профиль: [ссылка]
```

## Метрики успеха

**PWA:**
- Установок на домашний экран > 30% активных пользователей
- Retention после установки +40%
- Время загрузки < 2 сек
- Lighthouse PWA score > 90

**Админка:**
- Среднее время модерации < 24 часа
- Процент одобренных профилей > 85%
- Zero критических инцидентов без реакции > 1 часа
- Все платежи отслеживаемы

## Технические требования

### PWA setup:
```javascript
// manifest.json
{
  "display": "standalone",
  "orientation": "portrait",
  "start_url": "/",
  "scope": "/",
  "icons": [...]
}

// Service Worker
- Cache static assets
- Offline fallback page
- Skip waiting on update
```

### Filament Resources:
```php
- UserResource (CRUD пользователей)
- CoachProfileResource (модерация)
- WorkoutResource (управление)
- BookingResource (просмотр)
- PaymentResource (мониторинг)
- CityResource (справочник)
- SportResource (справочник)
```

### Dashboard widgets:
- StatsOverview (ключевые метрики)
- ModerationQueue (очередь)
- RecentPayments (последние платежи)
- RegistrationChart (график)
