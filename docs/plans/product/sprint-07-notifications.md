# Sprint 7: Уведомления и управление бронями

## Проблема / Краткое описание

Пользователи должны получать своевременные уведомления о важных событиях: напоминания о тренировках, информация о новых записях, возможность отмены. Атлетам нужна возможность отменить бронь с возвратом денег при соблюдении условий.

## Ценность для пользователя

**Для атлетов:** Не пропустить тренировку благодаря напоминаниям, возможность отменить планы с возвратом денег при уважительной причине.

**Для тренеров:** Быть в курсе записей на тренировки, планировать нагрузку, получать уведомления об отменах.

**Для бизнеса:** Снижение no-show, повышение лояльности через гибкую политику отмен, улучшение коммуникации.

## Скоуп спринта

### Включено:
- Email уведомления о всех ключевых событиях
- Напоминание за 2 часа до тренировки
- Отмена брони атлетом (>24 часа до начала)
- Возврат денег при отмене
- Уведомления тренеру о записях/отменах
- Автоматическое завершение прошедших тренировок
- Настройки уведомлений в профиле

### Исключено:
- Push-уведомления
- SMS-уведомления
- In-app уведомления
- Чат между тренером и атлетом
- Перенос брони на другую дату
- Частичный возврат при поздней отмене

## Критерии приёмки

### Отмена брони атлетом

**Дано:** Атлет с оплаченной бронью
**Когда:** Открывает детали брони в личном кабинете
**Тогда:**
1. Если до тренировки > 24 часов:
   - Видит кнопку "Отменить запись"
   - При клике → подтверждение
   - После подтверждения → возврат денег инициирован
   - Статус меняется на "Отменено"
   - Email об отмене отправлен
2. Если до тренировки < 24 часов:
   - Кнопка отмены недоступна
   - Текст: "Отмена невозможна менее чем за 24 часа"

### Процесс возврата

**Дано:** Атлет отменил бронь вовремя
**Когда:** Подтверждает отмену
**Тогда:**
1. Вызов Тинькофф Cancel API
2. Статус платежа → "Возвращается"
3. Email: "Возврат обработан, деньги поступят в течение 3-5 дней"
4. Место на тренировке освобождается
5. Тренер получает уведомление об отмене

### Напоминание о тренировке

**Дано:** Оплаченная бронь на тренировку
**Когда:** До начала остаётся 2 часа
**Тогда:**
- **Атлет получает email:**
  - Тема: "Напоминание: тренировка через 2 часа"
  - Детали: время, место, тренер
  - Ссылка на детали в личном кабинете
- **Тренер получает email:**
  - Список участников
  - Количество записавшихся
  - Ссылка на управление тренировкой

### Уведомление о новой записи

**Дано:** Атлет успешно оплатил место
**Когда:** Платёж подтверждён
**Тогда тренер получает email:**
- Тема: "Новая запись на вашу тренировку"
- Кто записался (имя)
- На какую тренировку
- Сколько мест занято/осталось

### Автозавершение тренировок

**Дано:** Тренировка с датой в прошлом
**Когда:** Прошло 2 часа после времени начала
**Тогда:**
- Статус автоматически → "Завершена"
- Тренировка исчезает из "Предстоящих"
- Переходит в "Прошедшие"
- В будущем: запрос отзыва от атлетов

### Настройки уведомлений

**Дано:** Пользователь в профиле
**Когда:** Открывает "Настройки уведомлений"
**Тогда может включить/выключить:**
- Напоминания о тренировках (по умолчанию ВКЛ)
- Новости и предложения (по умолчанию ВЫКЛ)
- Изменения в расписании (по умолчанию ВКЛ)
- Для тренера: новые записи (по умолчанию ВКЛ)

## UX заметки

### Отмена брони:
- Красная кнопка "Отменить запись"
- Модальное окно подтверждения
- Чёткое объяснение политики возврата
- Прогресс-индикатор при обработке

### Визуальные индикаторы времени:
- 🟢 Можно отменить (>24ч)
- 🔴 Отмена невозможна (<24ч)
- ⏰ Скоро начало (<2ч)

### Email дизайн:
- Брендированный header
- Чёткая структура информации
- CTA кнопки для действий
- Мобильная адаптация
- Unsubscribe link в футере

### Таймлайн отмены:
```
Создание брони → ... → За 24ч (последний шанс отменить) → За 2ч (напоминание) → Начало → +2ч (завершение)
```

## Валидация и бизнес-правила

### Политика отмены (финальное решение):
- **>24 часа до начала:** Полный возврат
- **<24 часа до начала:** Отмена запрещена, деньги не возвращаются
- Время считается по часовому поясу места тренировки

### Ограничения:
- Максимум 3 отмены в месяц (soft limit, предупреждение)
- Нельзя отменить уже начавшуюся тренировку
- Нельзя отменить завершённую тренировку
- Возврат только на ту же карту

### Background jobs:
- SendWorkoutReminderJob - за 2 часа
- CompleteWorkoutJob - через 2 часа после начала
- ProcessRefundJob - асинхронный возврат

## Метрики успеха

- Процент отмен < 15%
- Своевременная доставка напоминаний = 100%
- Успешные возвраты > 99%
- Открываемость email напоминаний > 60%
- No-show после напоминания < 5%

## Технические требования

### Email сервис:
- Транзакционные email через очереди
- Шаблоны в Blade/Vue
- Tracking открытий (опционально)
- Retry при неудаче

### Scheduler:
```php
// Напоминания каждые 10 минут
$schedule->job(SendRemindersJob::class)
         ->everyTenMinutes();

// Завершение тренировок каждый час
$schedule->job(CompleteWorkoutsJob::class)
         ->hourly();
```

### Refund через Тинькофф:
- Cancel API с amount
- Async обработка через job
- Status webhook handling
- Логирование всех операций
