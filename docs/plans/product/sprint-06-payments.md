# Sprint 6: Оплата и платежи

## Проблема / Краткое описание

Атлеты должны иметь возможность оплатить забронированное место банковской картой. Интеграция с Тинькофф Acquiring обеспечит безопасный приём платежей. После успешной оплаты место гарантировано закрепляется за атлетом.

## Ценность для пользователя

**Для атлетов:** Удобная и безопасная оплата картой, мгновенное подтверждение брони, электронный чек.

**Для тренеров:** Гарантия оплаты, автоматический учёт поступлений, отсутствие наличных расчётов.

**Для бизнеса:** Автоматизация платежей, прозрачность транзакций, основа для монетизации (комиссия в будущем).

## Скоуп спринта

### Включено:
- Интеграция с Тинькофф Acquiring
- Оплата банковской картой
- Экраны успешной/неуспешной оплаты
- Электронный чек на email
- История платежей в личном кабинете
- Обработка webhook от платёжной системы
- Автоматическое обновление статуса брони

### Исключено:
- Возвраты денег (refund) - будет в Sprint 7
- Альтернативные способы оплаты (ЮMoney, СБП)
- Сохранение карт для повторных платежей
- Подписки и рекуррентные платежи
- Разделение платежа (split payment)
- Оплата нескольких тренировок одновременно

## Критерии приёмки

### Переход к оплате

**Дано:** Атлет на странице подтверждения брони с таймером
**Когда:** Нажимает "Оплатить"
**Тогда:**
1. Создаётся платёжная сессия в Тинькофф
2. Редирект на платёжную форму Тинькофф
3. Видит страницу оплаты банка с суммой и описанием
4. Может ввести данные карты

### Успешная оплата

**Дано:** Атлет ввёл корректные данные карты
**Когда:** Платёж проходит успешно
**Тогда:**
1. Редирект обратно на split.fitness/payment/success
2. Видит экран "Оплата прошла успешно!"
3. Детали: тренировка, дата, место, сумма
4. Кнопка "Перейти к моим тренировкам"
5. Получает email с подтверждением и чеком
6. Бронь меняет статус на "Оплачено"

### Неуспешная оплата

**Дано:** Платёж отклонён банком
**Когда:** Атлет возвращается на сайт
**Тогда:**
1. Попадает на split.fitness/payment/fail
2. Видит сообщение "Оплата не прошла"
3. Причина (если известна): "Недостаточно средств" / "Карта отклонена"
4. Таймер брони продолжает идти
5. Кнопка "Попробовать другую карту"
6. Кнопка "Отменить бронь"

### Обработка webhook

**Дано:** Тинькофф отправляет уведомление о статусе платежа
**Когда:** Webhook приходит на /api/payments/webhook
**Тогда:**
1. Проверяется подпись запроса
2. Обновляется статус платежа в БД
3. Обновляется статус брони
4. Отправляются email уведомления
5. Ответ 200 OK банку

### История платежей

**Дано:** Атлет в личном кабинете
**Когда:** Открывает раздел "Платежи" или детали тренировки
**Тогда видит:**
- Список всех платежей с датами
- Статус: Успешно / Отменено / Возвращено
- Сумма и описание
- Ссылка на электронный чек
- Возможность скачать чек PDF

### Истечение платёжной сессии

**Дано:** Атлет не завершил оплату в течение 15 минут
**Когда:** Возвращается на сайт после истечения
**Тогда:**
- Бронь уже отменена
- Показывается сообщение о истечении времени
- Предложение записаться заново

## UX заметки

### Платёжный флоу:
1. Бронь → "Оплатить"
2. Редирект на Тинькофф
3. Ввод карты на стороне банка
4. 3D Secure (если требуется)
5. Возврат на сайт
6. Экран результата

### Экран успеха:
- ✅ Большая зелёная галочка
- Заголовок: "Оплата прошла успешно!"
- Детали тренировки в карточке
- "Мы отправили подтверждение на ваш email"
- Конфетти анимация (опционально)

### Экран неудачи:
- ❌ Красный крест
- Заголовок: "Оплата не прошла"
- Объяснение проблемы (если известно)
- Опции действий
- Оставшееся время брони

### Email с чеком:
**Тема:** "Чек об оплате - Split.fitness"
**Содержимое:**
- Номер платежа
- Дата и время
- Описание: "Тренировка [вид спорта] [дата]"
- Сумма
- Ссылка на электронный чек Тинькофф

## Валидация и безопасность

### Webhook security:
- Проверка signature (sha256 hash)
- IP whitelist (если поддерживается)
- Идемпотентность обработки
- Логирование всех запросов

### Статусы платежа:
- **created** - создан, ожидает оплаты
- **pending** - в обработке банком
- **succeeded** - успешно оплачен
- **failed** - отклонён
- **refunded** - возвращён (будущее)

### Бизнес-правила:
- Один платёж = одна бронь
- Нельзя оплатить уже оплаченную бронь
- Нельзя оплатить отменённую бронь
- Сумма платежа = цена слота из брони (не из запроса)

## Метрики успеха

- Конверсия переход к оплате → успешная оплата > 85%
- Время на оплату < 2 минуты
- Процент технических ошибок < 1%
- Процент успешной обработки webhook = 100%
- Доставка email с чеком > 99%

## Технические требования

### Тинькофф Acquiring API:
- Init - создание платежа
- GetState - проверка статуса
- Cancel - отмена (для возвратов)
- Webhook notification URL

### Хранение данных:
- НЕ хранить данные карт
- Хранить только payment_id от Тинькофф
- Логировать raw webhook payload
- Хранить ссылку на чек

### Тестовый режим:
- Тестовые карты Тинькофф
- Тестовый terminal_key
- Отдельный route для тест-среды
