# Product Plans

Продуктовые планы для проекта split.fitness, описывающие пользовательские требования и критерии приёмки для каждого спринта.

## Структура планов

Каждый продуктовый план описывает:
- **Проблему** - что решаем для пользователей
- **Ценность** - зачем это нужно каждой роли
- **Скоуп** - что входит и не входит в спринт
- **Критерии приёмки** - конкретные сценарии Given/When/Then
- **UX заметки** - важные детали интерфейса
- **Копирайт** - точные тексты и сообщения
- **Метрики успеха** - как измерить результат

## Ключевые решения

✅ **Принятые решения** (из plan-as-is.md):
- Политика отмены: возврат если >24 часа до тренировки
- Округление цены: округлять вверх (атлет платит чуть больше)
- Модерация тренеров: ручная проверка перед первой публикацией
- Платёжная система: Тинькофф Acquiring
- Напоминание: за 2 часа до тренировки

## Вне скоупа MVP

Не включено в текущие спринты:
- Отзывы и рейтинги
- Чат между тренером и атлетом
- Подписки / абонементы
- Реферальная программа
- Детальная аналитика для тренера
- Push-уведомления
- SMS-уведомления
- Социальная авторизация (OAuth)

## Связь с dev-планами

Каждый product план имеет соответствующий dev план в `docs/plans/dev/`:
- Product план описывает ЧТО делаем и ЗАЧЕМ
- Dev план описывает КАК реализуем технически
