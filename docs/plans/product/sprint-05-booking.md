# Sprint 5: Бронирование тренировок

## Проблема / Краткое описание

Атлеты должны иметь возможность записаться на выбранную тренировку с гарантией места. Необходима система резервирования, которая предотвращает двойные продажи и даёт время на оплату. Также нужен личный кабинет для управления записями.

## Ценность для пользователя

**Для атлетов:** Гарантия места на тренировке после записи, удобное управление своими бронированиями, история тренировок.

**Для тренеров:** Видимость записавшихся участников, планирование загрузки, отсутствие овербукинга.

**Для бизнеса:** Контроль за местами, предотвращение конфликтов, данные для аналитики.

## Скоуп спринта

### Включено:
- Процесс бронирования с резервацией места
- Таймер на оплату (15 минут)
- Личный кабинет атлета с записями
- Просмотр участников тренировки (для тренера)
- Автоматическое освобождение неоплаченных броней
- Предотвращение двойной записи
- История прошедших тренировок

### Исключено:
- Отмена брони атлетом (будет в Sprint 7)
- Перенос брони на другую дату
- Лист ожидания
- Покупка нескольких мест
- Приглашение друзей
- QR-код для check-in

## Критерии приёмки

### Запись на тренировку

**Дано:** Атлет просматривает карточку тренировки
**Когда:** Нажимает "Записаться"
**Тогда:**
1. Если не авторизован → редирект на вход/регистрацию
2. Если авторизован:
   - Проверка наличия свободных мест
   - Создание брони со статусом "Ожидает оплаты"
   - Резервирование места (уменьшение доступных мест)
   - Переход на страницу подтверждения
   - Запуск таймера 15 минут
3. Если мест нет → сообщение "К сожалению, места закончились"

### Страница подтверждения брони

**Дано:** Атлет успешно зарезервировал место
**Когда:** Попадает на страницу подтверждения
**Тогда видит:**
- Детали тренировки (что, где, когда)
- Информация о тренере
- Цена к оплате
- Таймер обратного отсчёта (14:59... 14:58...)
- Кнопка "Оплатить"
- Предупреждение: "Бронь будет отменена через X минут"

### Истечение времени брони

**Дано:** Атлет не оплатил в течение 15 минут
**Когда:** Таймер достигает 00:00
**Тогда:**
- Бронь автоматически отменяется
- Место освобождается для других
- Показывается сообщение "Время брони истекло"
- Предложение записаться заново (если есть места)

### Личный кабинет атлета

**Дано:** Авторизованный атлет
**Когда:** Открывает "Мои тренировки"
**Тогда видит вкладки:**
1. **Предстоящие:**
   - Оплаченные тренировки (отсортированы по дате)
   - Для каждой: дата, время, место, тренер, статус оплаты
   - Кнопка "Подробнее"
2. **Ожидают оплаты:**
   - Брони с таймером
   - Кнопка "Оплатить"
3. **Прошедшие:**
   - История посещённых тренировок
   - Возможность оставить отзыв (в будущем)

### Просмотр участников (для тренера)

**Дано:** Тренер открывает свою тренировку
**Когда:** Смотрит раздел "Участники"
**Тогда видит:**
- Список записавшихся: имя, фото, статус оплаты
- Количество: "Записано 7 из 10"
- Индикация: ✓ Оплачено / ⏱ Ожидает оплаты
- Контакты участников (телефон для связи)

### Предотвращение двойной записи

**Дано:** Атлет уже записан на тренировку
**Когда:** Пытается записаться повторно
**Тогда:**
- Кнопка "Записаться" неактивна
- Показан текст "Вы уже записаны"
- Ссылка на детали брони в личном кабинете

## UX заметки

### Процесс бронирования:
1. Карточка тренировки → "Записаться"
2. Подтверждение брони → таймер + детали
3. Оплата (следующий спринт)
4. Успех → личный кабинет

### Таймер оплаты:
- Крупные цифры, заметное расположение
- Красный цвет при < 5 минут
- Анимация последних 60 секунд
- Звуковой сигнал при истечении (опционально)

### Статусы брони в личном кабинете:
- 🟡 **Ожидает оплаты** (с таймером)
- 🟢 **Оплачено**
- 🔴 **Отменено**
- ⚫ **Истекло**

### Мобильная версия:
- Фиксированная кнопка "Записаться" внизу экрана
- Свайп для отмены неоплаченной брони
- Компактные карточки в личном кабинете

## Валидация и бизнес-правила

### При бронировании:
- Нельзя записаться на прошедшую тренировку
- Нельзя записаться дважды на одну тренировку
- Проверка доступности мест с блокировкой (транзакция)
- Максимум 3 неоплаченные брони одновременно

### Резервирование места:
- Атомарная операция в БД с блокировкой
- При конфликте → повтор через 100ms (максимум 3 попытки)
- При неудаче → сообщение "Попробуйте ещё раз"

### Автоочистка:
- Проверка истёкших броней каждую минуту
- Освобождение мест для броней старше 15 минут
- Уведомление атлета об истечении (email)

## Копирайт и тексты

### Подтверждение брони:
**Заголовок:** "Место зарезервировано!"
**Текст:** "У вас есть 15 минут, чтобы завершить оплату"
**Таймер:** "Осталось: 14:52"
**Кнопка:** "Оплатить 500 ₽"

### Истечение брони:
**Заголовок:** "Время брони истекло"
**Текст:** "К сожалению, ваше место было передано другому участнику"
**Кнопка:** "Попробовать снова" (если есть места)

### Двойная запись:
**Сообщение:** "Вы уже записаны на эту тренировку"
**Ссылка:** "Перейти к деталям"

### Нет мест:
**Заголовок:** "Места закончились"
**Текст:** "Пока вы оформляли запись, последнее место было забронировано"
**Кнопка:** "Найти другую тренировку"

### Email о истечении:
**Тема:** "Ваша бронь на тренировку отменена"
**Текст:**
```
К сожалению, время оплаты вашей брони истекло.
Тренировка: [название]
Дата: [дата и время]

Если места ещё есть, вы можете записаться снова.
```

## Метрики успеха

- Конверсия "Записаться" → создание брони > 90%
- Процент оплаченных броней > 70%
- Среднее время до оплаты < 5 минут
- Процент истёкших броней < 30%
- Отсутствие случаев овербукинга (0 инцидентов)

## Технические требования

### Транзакционная безопасность:
- `SELECT ... FOR UPDATE` при проверке мест
- Атомарное увеличение `slots_booked`
- Rollback при любой ошибке

### Background jobs:
- `ExpirePendingBookingsJob` - каждую минуту
- Асинхронная отправка email уведомлений

### API endpoints:
- `POST /api/bookings` - создание брони
- `GET /api/athlete/bookings` - список броней
- `GET /api/coach/workouts/:id/participants` - участники
