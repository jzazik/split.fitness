# Sprint 3: Создание и управление тренировками

## Проблема / Краткое описание

Тренеры с одобренными профилями должны иметь возможность создавать тренировки, указывать место проведения на карте, устанавливать цену и количество мест. Тренировки должны публиковаться на карту для привлечения атлетов.

## Ценность для пользователя

**Для тренеров:** Возможность создавать и управлять своими тренировками, устанавливать удобное расписание, контролировать количество участников и доход.

**Для атлетов:** Появление тренировок на карте для записи (подготовка к следующему спринту).

**Для бизнеса:** Наполнение платформы контентом, создание предложения для атлетов.

## Скоуп спринта

### Включено:
- Создание тренировки с выбором места на карте
- Установка даты, времени и длительности
- Указание общей стоимости и количества мест
- Автоматический расчёт цены за слот
- Сохранение черновиков
- Публикация тренировок (только для одобренных тренеров)
- Список своих тренировок
- Редактирование неопубликованных тренировок
- Отмена тренировки (если нет броней)

### Исключено:
- Повторяющиеся тренировки
- Шаблоны тренировок
- Групповое редактирование
- Импорт расписания
- Автоматическая публикация по времени
- Уведомления участникам при изменениях

## Критерии приёмки

### Создание тренировки

**Дано:** Тренер с одобренным профилем
**Когда:** Нажимает "Создать тренировку"
**Тогда:**
1. Открывается форма создания с полями:
   - Вид спорта (из профиля тренера)
   - Название (опционально)
   - Описание (опционально)
   - Дата и время начала
   - Длительность (30, 45, 60, 90, 120 минут)
   - Место на карте (обязательно)
   - Адрес/описание места
   - Общая стоимость тренировки
   - Количество мест
2. Показывается расчёт: "Цена для атлета: XXX ₽"
3. Можно сохранить как черновик
4. Можно сразу опубликовать

### Выбор места на карте

**Дано:** Тренер заполняет форму тренировки
**Когда:** Нажимает "Указать место на карте"
**Тогда:**
1. Открывается карта города
2. Может поставить маркер в нужном месте
3. Автоматически определяется адрес (если возможно)
4. Может вручную уточнить описание места
5. Подтверждает выбор

### Расчёт цены слота

**Дано:** Тренер указал общую стоимость и количество мест
**Когда:** Вводит значения
**Тогда:**
- Автоматически показывается цена за одно место
- Если деление не точное, округляется вверх
- Пример: 1000₽ / 3 места = 334₽ за место
- Показывается итог: "Вы получите 1002₽ (334₽ × 3)"

### Публикация тренировки

**Дано:** Тренер создал черновик тренировки
**Когда:** Нажимает "Опубликовать"
**Тогда:**
- Если профиль одобрен → тренировка публикуется
- Если профиль на модерации → показывается сообщение "Дождитесь одобрения профиля"
- Опубликованная тренировка сразу появляется на карте
- Статус меняется на "Опубликовано"

### Список тренировок

**Дано:** Тренер в личном кабинете
**Когда:** Открывает "Мои тренировки"
**Тогда:**
- Видит список всех своих тренировок
- Фильтры: Все / Черновики / Опубликованные / Прошедшие
- Для каждой тренировки показано:
  - Дата и время
  - Место
  - Записано людей / всего мест
  - Статус
  - Кнопки действий

### Редактирование тренировки

**Дано:** Тренер хочет изменить тренировку
**Когда:** Нажимает "Редактировать"
**Тогда:**
- Если есть оплаченные брони → редактирование запрещено
- Если броней нет → можно изменить все поля
- После сохранения изменения применяются

### Отмена тренировки

**Дано:** Тренер хочет отменить тренировку
**Когда:** Нажимает "Отменить тренировку"
**Тогда:**
- Если есть оплаченные брони → отмена запрещена, показать сообщение
- Если броней нет → подтверждение отмены
- После отмены тренировка исчезает с карты
- Статус меняется на "Отменена"

## UX заметки

### Форма создания тренировки:
- Разбита на логические блоки: Что → Когда → Где → Сколько
- Валидация в реальном времени
- Автосохранение черновика каждые 30 секунд
- Прогресс-бар заполнения формы

### Карта для выбора места:
- По умолчанию центрирована на городе тренера
- Поиск по адресу (опционально)
- Возможность переместить маркер
- Поле для уточнения: "Дополнительные ориентиры"

### Список тренировок:
- Карточки с ключевой информацией
- Цветовое кодирование статусов
- Quick actions: дублировать, отменить, поделиться
- Сортировка по дате (ближайшие сверху)

### Мобильная версия:
- Вертикальная форма с шагами
- Полноэкранная карта при выборе места
- Свайпы для быстрых действий в списке

## Валидация полей

### При создании тренировки:
- **Дата:** Не раньше завтрашнего дня, не позже чем через 3 месяца
- **Время:** Кратно 15 минутам (08:00, 08:15, 08:30...)
- **Общая стоимость:** Минимум 100₽, максимум 50000₽
- **Количество мест:** От 1 до 50
- **Место на карте:** Обязательно
- **Описание места:** Минимум 10 символов

### Бизнес-правила:
- Нельзя создать 2 тренировки в одно время и место
- Минимум 30 минут между тренировками одного тренера
- Максимум 10 активных тренировок одновременно (на старте)

## Статусы тренировки

- **draft** - черновик, виден только тренеру
- **published** - опубликована, видна на карте
- **cancelled** - отменена тренером
- **completed** - прошедшая (автоматически через 2 часа после начала)

## Копирайт и тексты

### Пустое состояние:
**Заголовок:** "Создайте свою первую тренировку"
**Текст:** "Опубликуйте тренировку на карте и начните принимать записи от атлетов"
**Кнопка:** "Создать тренировку"

### Форма создания:
**Заголовок блоков:**
- "Что за тренировка?"
- "Когда проводим?"
- "Где встречаемся?"
- "Стоимость и места"

### Сообщения при публикации:
**Успех:** "Тренировка опубликована и уже доступна на карте!"
**Модерация:** "Дождитесь одобрения вашего профиля, чтобы публиковать тренировки"
**Ошибка:** "Проверьте заполнение всех обязательных полей"

### Подтверждение отмены:
**Заголовок:** "Отменить тренировку?"
**Текст:** "Тренировка будет удалена с карты. Это действие нельзя отменить."
**Кнопки:** "Да, отменить" / "Оставить"

## Метрики успеха

- Среднее время создания тренировки < 5 минут
- Процент опубликованных от созданных > 80%
- Количество тренировок на тренера в неделю > 3
- Процент заполненных описаний > 60%
